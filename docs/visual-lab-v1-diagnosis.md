# Visual Lab V1 — 诊断报告

## 检测方式

Playwright 全视口 DOM + CSS 属性扫描 + 3 个鼠标位置截图

## 发现的 5 个视觉故障

| # | 问题 | 元素 | 原因 |
|---|------|------|------|
| 1 | 人物头顶在视口外 | img[0] top=-253px | `h-[80vh] object-bottom` + hero height 100vh 导致图片向上溢出 |
| 2 | PAN YIBING 后层不可见 | 描边文字 opacity 0.07 | `rgba(200,192,184,0.07)` 在深黑背景上几乎完全透明 |
| 3 | PAN YIBING 前层只剩一半 | clipPath: inset(45% 0 0 0) | 刻意切掉上半部分作为"穿插效果"，实际造成阅读困难 |
| 4 | Hero overflow:hidden | section 容器 | 人物视频在边界处被裁切 |
| 5 | 抽象占位卡片 | 4 个纯色块 | 没有使用真实项目封面 |

## 根因总结

V1 的"穿插"效果依赖三个不稳定技术：低透明度描边文字、clipPath 裁剪、多层 z-index 堆叠。这些在实际渲染中产生的是"文字看不见 + 人物被裁切"，而非预期的"杂志封面穿插"。
